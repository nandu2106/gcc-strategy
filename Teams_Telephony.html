End-to-End Call Flow

Internal Call: Teams User A â†’ Teams Cloud â†’ User B (local Teams) | External Call: Teams User â†’ PSTN Gateway (Operator Connect/Calling Plans) â†’ External PSTN

1. Initiation

User dials number in Teams client. Identity verified via Azure AD. Calling policy checked.

2. Routing

Call routed to destination. Internal: direct to user. External: to PSTN gateway (region-specific).

3. Connection

Call established. Media streams flow (encrypted). Call quality monitored in real-time.

4. Recording

If required: recording starts automatically. Compliance policies enforced. Legal hold applied if needed.

5. Disconnect

Call ends. Recording stopped. CDR (call detail record) logged. Audit trail immutable.

6. Analytics

Call metrics analyzed (MOS, latency). Trends reported. Optimization opportunities identified.
