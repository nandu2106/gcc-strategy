Risks & mitigations

AI bias / drift (forecasting)

Impact: planning errors, margin risk

Mitigation: bias audits, drift monitoring, human override, diverse training data.

Privacy & regulatory drift

Impact: compliance exposure, reputational risk

Mitigation: privacy-by-design, DLP, DPIAs, legal review loops.

Runaway GPU / token spend

Impact: cost volatility

Mitigation: quotas, hard caps, auto-suspend, FinOps anomaly alerts.

Vendor lock‑in

Impact: reduced flexibility & leverage

Mitigation: abstraction layer, portable formats, periodic “exit drills”.

Portfolio governance

  • Go/No‑Go gates: Concept (≤6 weeks) → Build (≤3 months) → Scale (≤6 months) → Run (ongoing).

  • Kill-switch: If value is not material after 2 review cycles, optimize or sunset.

AI vendor abstraction

  • Model gateway: One orchestration layer routes to approved models (policy + cost + quality).
  • Portable prompts: Standard prompt patterns; avoid vendor-specific tooling lock-in.
  • Evaluation cadence: Quarterly bake-off across cost, latency, quality, safety.

AI governance controls

Prompt & response logging

  • Log request/response (policy-controlled).
  • Quarterly audit for leakage.
  • Prompt-injection detection.

Model approval

  • Risk review before production.
  • Bias testing for decision-support.
  • Rollback plan required.

Red teaming

  • Adversarial testing each quarter.
  • Jailbreak monitoring.
  • Output filters for brand/legal.

Privacy handling

  • Tokenize PII.
  • Access via least privilege.
  • Data residency enforced.

Guardrails that matter

  • Hard caps: auto-suspend at 80% of budget.
  • Idle cleanup: stop resources after 7 days idle.
  • Anomaly detection: alert on unusual spend spikes and auto-throttle lane 4.
  • Showback: weekly dashboards; chargeback above agreed thresholds.

AI-specific failure modes

Model endpoint unavailable

  • Fallback to cached responses (time-bounded)
  • Failover to secondary endpoint

Data pipeline stale

  • Freeze predictions with warning
  • Use last-known-good dataset with age label

External LLM outage

  • Switch to secondary provider/model
  • Queue non-critical requests

Edge device failure

  • Dual edge nodes for critical lines
  • Manual override procedures
